# PRD：旺财 Web 自动化控制台（V1.2）

## 1. 产品名称
旺财（WangCai）- 超强财务报表数据自动化控制台

## 2. 产品目标
1. 面向非技术财务用户，提供全网页按钮化操作。
2. 从数据导入到报表生成、审批发布，实现单页面闭环。
3. 降低导入门槛：支持云端取数 + 本地乱表智能识别导入。
4. 提供可视化状态与预警，并可直接打开报表文件夹或网页预览 Excel。

## 3. 目标用户
1. 财务会计：导入数据、生成初步报表。
2. 财务经理：审批后发布看板。
3. 管理层：查看结果与波动预警。

## 4. 页面结构
1. 顶部头部：品牌名“旺财”、副标题、状态标签、头图。
2. 状态卡片：审批状态、初步报表路径（省略显示+悬浮完整路径）。
3. 一键操作卡片：开始月度结账、查看审批状态、生成本月初步报表、审批通过更新看板。
4. 导入卡片A：云端取数。
5. 导入卡片B：本地表单导入。
6. 执行结果卡片：显示最后动作、时间和简明结果。

## 5. 功能需求

### F1 云端取数
输入支持：
- 股票代码（如 IBM）
- ERP 账套名称
- API 地址（JSON/CSV）

处理逻辑：
1. 股票代码：执行 01_extract 脚本抓取并更新数据库。
2. API URL：优先按 JSON 解析入库，失败再按 CSV 尝试。
3. ERP账套名：提示用户上传该账套导出的文件或提供 API 地址。

### F2 本地表单导入（CSV/Excel）
处理逻辑：
1. 标准格式文件：直接导入数据库。
2. 非标准格式文件：进入“智能字段确认”。
   - 系统给出自动识别结果（日期列、金额列等）
   - 用户确认/修改后导入
3. 导入成功后，数据统一落地为标准长表结构并写入数据库。

### F3 一键流程控制
1. 开始月度结账：根据审批状态自动分流 N08/N09。
2. 查看审批状态：输出下一步操作建议。
3. 生成本月初步报表：执行 N08 并回显文件路径。
4. 审批通过，更新看板：写入审批信号并执行 N09。

### F4 报表路径交互
1. 路径展示为蓝色可点击链接。
2. 支持“打开报表文件夹”。
3. 支持“网页预览 Excel”。

## 6. 业务规则
1. 审批通过判定：审批状态为 Approved 才可发布看板。
2. 收入环比绝对值超过 10% 时标记预警。
3. 数据准备失败自动重试一次。

## 7. 交互与视觉规范
1. 主题为暖黄渐变，风格专业且亲和。
2. 状态/预警使用高对比颜色。
3. 报表路径长文本使用省略号，悬浮展示完整路径。
4. 页面布局整齐：状态卡与操作卡同排同高。

## 8. 非功能要求
1. 本地可直接启动运行。
2. 公网部署支持 GitHub + Web Service（如 Render/Railway）。
3. 关键动作保留日志用于追溯。

## 9. 验收标准
1. 页面可打开并完整渲染头图与各卡片。
2. 云端取数可识别股票代码/API URL/ERP名称输入。
3. 本地乱表上传后会进入字段确认流程。
4. “status.txt”不在当前状态卡片显示。
5. 报表路径可点击，并支持打开文件夹与网页预览。
6. 一键流程四按钮均可执行并回显结果。

## 10. 上线说明
1. 代码托管 GitHub。
2. 以 Python Web Service 方式部署。
3. 用户通过网页 URL 直接使用，无需命令行。
